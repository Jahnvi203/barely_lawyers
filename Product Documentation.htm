About the Documentation

Purpose

 

The purpose of this documentation is to help Community Justice Centre (CJC) understand the web app developed by 26_CL_BA_Barely Lawyers_Justice is Managed as part of IS483 IS/SMT Project Experience at Singapore Management University in collaboration with CJC.

Product Definition

Product 1

 

Preliminary Intake Assessment for Maintenance and Divorce for the user as well as the Maintenance Dashboard and the Divorce Dashboard as part of the Admin Dashboard for CJC

 

Product 2

 

OSLAS Eligibility Assessment

 

Product 3

 

Prescriptive Analysis for prescribe Case Advice for new Cases based on previous Cases data

 

Product 4

 

Content Management System to update the questions and their details for the Preliminary Intake Assessment and the OSLAS Eligibility Assessment

User Journey

Product 1

For Preliminary Intake Assessment (for users)

 

  1. Navigate to the home page at https://jahnvi371.pythonanywhere.com/

  1. To navigate to the Maintenance PIA, in the left side navigation menu, click on Family Matters and click on Maintenance

  1. To navigate to the Divorce PIA, in the left side navigation menu, click on Family Matters and click on Divorce

  1. On clicking on Maintenance or Divorce, the user will be directed to the questions in the respective PIA

  1. Questions can be a mix of single choice and multiple choice and the options for the questions can range from 2 to 4
    1. For a single choice question, the user can only select 1 option

    1. For a multiple choice question, the user can select more than 1 option

  1. To go to the next question, the user must select the relevant option(s) for the current question and click on the Next > button

  1. To go to the previous question, the user must click on the < Back button from the current question

  1. Where a question has a glossary, the user can access the glossary by clicking on Need More Information?

The glossary will appear as a right side navigation bar with an overlay

  1. Where a question requires user input(s) (referred in the Admin Dashboard as Additional Inputs), the user must fill in the user input(s) before proceeding to the next question

  1. A question can have information in addition to the question itself and its options but such additional information is not a glossary (referred in the Admin Dashboard as Additional Annex)

  1. Once a user completes the questionnaire based on their responses to the questions asked, they will be shown their responses to the questions asked along with the relevant help and resources

  1. Once a user completes the questionnaire, the user can download their responses to the questions asked along with the relevant help and resources by clicking on download a PDF. A sample of this report can be found at https://jahnvi371.pythonanywhere.com/report/113749.pdf
  2. Once a user completes the questionnaire, the user can also email their responses to the questions asked along with the relevant help and resources by clicking on email a PDF to yourself. The user will be prompted to enter their email address. Upon filling in their email address and clicking on Submit, the user will receive the email. A sample of this email can be found below

Product 4

For Preliminary Intake Assessment

 

  1. Navigate to the Admin Dashboard page at https://jahnvi371.pythonanywhere.com/admin/dashboard

  1. To navigate to the Maintenance CMS, in the left side navigation menu, click on Family Matters and click on Maintenance CMS. A glimpse of the Maintenance CMS can be found below

  1. To navigate to the Divorce CMS, in the left side navigation menu, click on Family Matters and click on Divorce CMS. A glimpse of the Divorce CMS can be found below

  1. To edit the details of a question, click on the expand/collapse icon for the question. You will see the options for the question then click on Edit

  1. You will be directed to a page with the details of the question displayed in editable inputs

  1. To edit the question itself, click on the input on the right of Question
  2. The questions type, which could be scq standing for single-choice question or mcq standing for multiple-choice question, is not editable
  3. To edit the options of the question, click on the inputs on the right of Option 1, Option 2, Option 3 and Option 4 respectively
  4. To edit the images for the options of the question, click on the inputs on the right of Image URL for Option 1, Image URL for Option 2, Image URL for Option 3 and Image URL for Option 4 respectively
  5. To edit the glossary of the question, click on the input on the right of Glossary. This is a rich text editor so you can format the text, add lists and tables, insert web links, images and videos or you can also add in custom HTML code
  6. To edit the additional annex of the question, click on the input on the right of Additional Annex. This is a rich text editor so you can format the text, add lists and tables, insert web links, images and videos or you can also add in custom HTML code
  7. Once you have edited the details of the question, click on Submit

  1. To edit an annex, click on the Edit button for the question

  1. You will be directed to a page with the annexs HTML code displayed in an editable input

  1. You can edit the annexs HTML code and click on Preview to preview the edited annex

  1. Once you have edited the annex, click on Submit
